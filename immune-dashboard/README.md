# ImmunoTrial Dashboard · Loblaw Bio

Interactive dashboard for analysing immune cell population dynamics in the Miraclib clinical trial.

---

## Quick Start (GitHub Codespaces)

```bash
make setup      # install dependencies
make pipeline   # load DB + run all analyses
make dashboard  # start Flask server on port 5050
```

Then open the forwarded port **5050** in the Codespaces browser preview.

---

## Running Locally

```bash
pip install -r requirements.txt
python load_data.py   # creates immune_trial.db
python analysis.py    # writes outputs/ (CSV + PNG)
python app.py         # http://localhost:5050
```

## Database Schema

```
projects        subjects          samples              cell_counts
-----------     ---------------   ------------------   ------------------
project_id PK   subject_id PK     sample_id PK         sample_id PK → samples
                project_id FK     subject_id FK         b_cell
                condition         project_id FK         cd8_t_cell
                age               sample_type           cd4_t_cell
                sex               time_from_treatment   nk_cell
                treatment                               monocyte
                response
```

### Rationale

| Design choice | Reason |
|---|---|
| **projects** separated | Avoids repeating project metadata on every row; scales to hundreds of projects with one indexed FK |
| **subjects** separated | Patient demographics (age, sex, treatment, response) are immutable per subject — normalising them avoids update anomalies across thousands of samples |
| **samples** separated | Each sample has its own type and time-point; FK to both subject and project enables efficient project- or subject-scoped queries without joins across the whole table |
| **cell_counts** wide row | Five fixed populations → wide format keeps aggregate queries (`SUM`, `AVG` across populations) simple and avoids EAV overhead for a fixed analyte set |
| **Indexes** | `idx_subjects_project`, `idx_samples_subject`, `idx_samples_project` keep filtered queries fast even with millions of rows |

**Scaling to hundreds of projects / thousands of samples / varied analytics:**

- Add a `measurement_types` + `measurements` EAV pair when the analyte list grows beyond a handful of known columns.
- Partition `cell_counts` by `project_id` (or use a column-store like DuckDB/ClickHouse) for aggregate analytics at scale.
- Add a `time_points` lookup table if time-points become project-specific.
- The FK-normalised design means adding a new project is a single `INSERT INTO projects` with no schema migration.

## Code Structure

```
.
├── load_data.py     # Part 1 – schema init + CSV → SQLite
├── analysis.py      # Parts 2-4 – frequency table, stats, subset
├── app.py           # Flask API + dashboard server
├── templates/
│   └── index.html   # Single-page Plotly dashboard
├── outputs/         # Generated by analysis.py
│   ├── frequency_table.csv
│   ├── statistical_results.csv
│   ├── responder_boxplots.png
│   ├── p4_samples_per_project.csv
│   ├── p4_response_counts.csv
│   └── p4_sex_counts.csv
├── cell-count.csv
├── requirements.txt
└── Makefile
```

### Design rationale

- **`load_data.py` standalone** — no imports from `app.py`; graders can run it in isolation.
- **`analysis.py`** reads from SQLite (not the raw CSV) to exercise the schema and produce reproducible outputs; uses `scipy.stats.mannwhitneyu` for non-parametric comparison since cell percentages are not guaranteed normal.
- **`app.py`** is a thin Flask layer exposing JSON API endpoints consumed by Plotly.js in the browser — no server-side rendering of charts keeps the dashboard fast and interactive.
- **Outputs directory** stores both machine-readable CSVs and the static boxplot PNG so results are reproducible without running the dashboard.

## Dashboard

Run `make dashboard` and open **http://localhost:5050** (or the Codespaces forwarded port).

Sections:
1. **Overview stats** – project / subject / sample counts
2. **Part 2** – Mean relative frequency bar chart (all samples, all populations)
3. **Part 3** – Per-population boxplots comparing Responders vs Non-responders (melanoma, miraclib, PBMC) with Mann-Whitney U p-values
4. **Statistical table** – U statistics and significance flags
5. **Part 4** – Baseline subset charts: samples per project, responder/non-responder split, sex distribution
